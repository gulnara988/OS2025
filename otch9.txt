 Работа с пайпом .Часть 1.
	Обьявление массива из ддвух целых значений для хранений файловых дискрипторов,ассоциированных с пайпом:
	int fd[2];

        Создание пайпа(для связи между родственными процессами )
        pipe(fd)
	После выполнения pipe(fd) в массиве записываються знаечения(новые для каждого нового пайпа)

	Запись в пайпбайтовой строки:
        size = write(fd[1], string, 20);

	Чтение из пайпа байтовой строки 
        size = read(fd[0], resstring, 20);
	При чтении информации из пайпа прочитанные байты исчезают из каналасвязи

	Закрытие выходного9открытого ранее на запись)
        close(fd[1] 

	Закрытые входного потока (открытого ранее на чтение)
        close(fd[0])

Часть 2 
Двунаправленнная связь между родсвенными процессами

Для организации двунаправленной связи проще всего использовать два отдельных канала - pipe1 ждя передачи данных от родителя к ребенку и pipe2 для передачи данных от ребенка к родителю

pipe1 доступен через массив файловых дискрипторов fd1, а pipe2 - xthtp fd2

Пример выполнения программы

строка для дочернего процесса записанна в пайп1
Процесс-ребенок начинает чтенпиестроки из пайп2...
Сообщение от родителя: Информация для дочернего процесса от процесса-родителя
Процесс-ребенок начинает запись строки из пайп2...
Дочернему процесса не удалось записать строку в ппайп2
Родительский процесс читает информацию из pipe2 ...Сообщение от ребенка:
Родительский процесс завершил расначала работает процесс-родитель ,который записывает  информацию в пайп1 для процесса - ребенка
Затем родитель пытается прочитать информацию оттддочернего процеса,но в пайп2 информации еще нет ,поэтому родитель блокируется до того момента пока в канале не появится информация
процесс-ребенок читает строку из пайп1 записывает информацию в пайп2 и завершает работу.
Процесс родитель переключается из состояния ожидания в состояние "готовность", затем - в состояние "выполнение" и читает строку из пайпа2,после чего завершает работу.
